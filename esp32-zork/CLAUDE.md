# ESP32 Zork — Claude Notes

## What This Is

Z-machine text adventure server. The ESP32-S3 runs a Z-machine interpreter;
a phone browser connects via WebSocket and acts as the terminal. No extra
hardware required beyond the N16R8 board.

## Project Layout

PlatformIO project. Source in `src/`; web UI and story files in `data/`
(served from LittleFS). Future: frotz interpreter sources in `lib/frotz/`.

## Flashing

```
flash.bat          → compile + flash firmware (pio run -e dev -t upload)
flash.bat fs       → upload LittleFS filesystem (data/ directory)
flash.bat all      → flash firmware then filesystem
```

Always run `flash.bat fs` after changing `data/index.html` or adding story files.

## Serial Monitoring

UART0 on COM5 (CH343 bridge). Monitor at 115200 baud.
On boot, the IP address and mDNS name are printed. They repeat every 5 s.

## Access

- IP: printed on boot and every 5 s via serial
- mDNS: http://zork.local/ (works on most phones and Macs; Windows needs
  Bonjour or a browser that supports mDNS)

## WebSocket Protocol

`/ws` endpoint. Text frames only.

- **Client → Server (first message):** story filename, e.g. `zork1.z3` — starts the interpreter
- **Client → Server (subsequent):** one line of player input (no newline)
- **Server → Client:** interpreter output as raw text (may contain `\r\n`)

The browser terminal echoes the player's input locally before sending,
so the server never needs to echo it back.

`GET /api/games` returns a JSON array of story filenames present in LittleFS,
e.g. `["zork1.z3","hitchhiker.z5"]`. Only `.z1`–`.z9` files are included.

## Setup for Milestone 2 (one-time)

### 1. Get frotz sources

Run from the `esp32-zork/` directory:

```
get-frotz.bat
```

This clones frotz from gitlab.com/DavidGriffith/frotz, copies `src/common/`
into `lib/frotz/common/`, then removes the clone.

### 2. Add a story file

Copy a .z5 or .z3 story file into `data/` as `zork1.z5`.
Zork 1 (.z3) is in the public domain from the Interactive Fiction Archive.
The STORY_FILE constant in `src/main.cpp` points to `/littlefs/zork1.z5`.

### 3. Flash

```
flash.bat fs     -- upload data\ (index.html + story file) to LittleFS
flash.bat        -- compile and flash firmware
```

### Partition Note

Default partition scheme gives ~1.5 MB to LittleFS — enough for index.html
and one small story file. Before adding multiple files or a .z8:

Add to `platformio.ini` under `[env:dev]`:
```
board_build.partitions = default_16MB.csv
```

This gives ~9 MB to LittleFS. Re-run `flash.bat all` after changing it.

## Interpreter Architecture

```
Core 0 (AsyncWebServer task)        Core 1 (frotz task — "frotz")
─────────────────────────────       ─────────────────────────────
WiFi / TCP / HTTP                   zork_interpreter_task()
WebSocket event handler               os_init_setup / os_process_arguments
  WS_EVT_DATA →                       init_buffer/err/memory/process/sound
    xQueueSend(zork_input_q)          os_init_screen / init_undo / z_restart
  WS_EVT_CONNECT →                    interpret()  ← main loop
    xTaskCreatePinnedToCore             os_read_line() blocks on zork_input_q
  WS_EVT_DISCONNECT →                 os_display_char() → out_buf
    vTaskDelete(zork_task_handle)       out_flush() → send_to_client()
loop() → ws.cleanupClients()                         → ws.text(clientId, …)
```

Key files:
- `lib/frotz/zork_frotz.h` — shared types, globals, interpreter task declaration
- `lib/frotz/zork_os.cpp`  — all os_* implementations + zork_interpreter_task()
- `lib/frotz/common/`      — unmodified frotz interpreter core (user supplies)
- `src/main.cpp`           — WiFi/WebSocket setup + send_to_client callback

Story file access: frotz's init_memory() calls fopen(f_setup.story_filename).
LittleFS registers with ESP32 VFS at /littlefs/, so fopen("/littlefs/zork1.z5")
works with standard C stdio. No custom file I/O layer required.

## Potential Compile Issues

frotz common/ files may include `config.h` (generated by autoconf on Unix).
If the build fails with "config.h not found", create `lib/frotz/common/config.h`
with just:
```c
/* stub config.h for ESP32 Arduino build */
```

Some os_* signatures or f_setup field names may differ across frotz versions.
Check frotz.h for exact names and adjust zork_os.cpp if needed.
